# Web UI Internationalization (i18n)

## Overview

The Mayu Web UI uses Angular's built-in i18n system to support multiple languages. The source locale is English (`en`), with Japanese (`ja`) as the first supported translation.

## Architecture

- **Build-time i18n**: Angular compiles separate bundles per locale at build time
- **Source locale**: English (template text serves as source)
- **Translation format**: XLIFF 1.2 (`.xlf` files)
- **Translation files**: `ui/src/locale/messages.ja.xlf`

## File Structure

```
ui/
├── angular.json              # i18n configuration
├── src/
│   ├── locale/
│   │   ├── messages.xlf      # Extracted source messages (generated)
│   │   └── messages.ja.xlf   # Japanese translations (maintained manually)
│   └── app/
│       └── ...               # Components with i18n markers
```

## Configuration

In `angular.json`:

```json
{
  "projects": {
    "mayu": {
      "i18n": {
        "sourceLocale": "en",
        "locales": {
          "ja": "src/locale/messages.ja.xlf"
        }
      },
      "architect": {
        "build": {
          "options": {
            "localize": true
          },
          "configurations": {
            "development": {
              "localize": false
            }
          }
        }
      }
    }
  }
}
```

- Production builds generate bundles for all locales (`en` + `ja`)
- Development builds skip localization for faster iteration

## How to Mark Text for Translation

### Static text in templates

```html
<h3 i18n="@@vulnDetail.details">Details</h3>
<button i18n="@@vulnList.clearFilters">Clear filters</button>
```

### Text with interpolation

```html
<p i18n="@@pagination.showing">
  Showing <span class="font-medium">{{ startItem() }}</span>
  to <span class="font-medium">{{ endItem() }}</span>
  of <span class="font-medium">{{ total() }}</span> results
</p>
```

### Attributes

```html
<input
  placeholder="CVE-2024-..."
  i18n-placeholder="@@vulnList.filterIdPlaceholder"
/>
<button aria-label="Close menu" i18n-aria-label="@@sidebar.closeMenu">
```

### TypeScript code

```typescript
// Simple string
const label = $localize`:@@sidebar.nav.vulnerabilities:Vulnerabilities`;

// With interpolation
const msg = $localize`:@@vulnDetail.notFound:Vulnerability "${id}" not found.`;
```

## Custom ID Convention

All i18n markers must use custom IDs with `@@` prefix:

- Format: `@@{component}.{purpose}`
- Component names: `sidebar`, `header`, `pagination`, `vulnList`, `vulnDetail`
- Purpose: camelCase description of the text's role

Examples:
- `@@sidebar.appName` — App name in sidebar
- `@@vulnList.colSummary` — "Summary" column header
- `@@vulnDetail.backToList` — Back navigation link
- `@@pagination.pageOf` — "Page X of Y" text

## Translation Workflow

### Adding new translatable text

1. Add `i18n="@@customId"` to the template element (or `$localize` in TS)
2. Extract messages:
   ```bash
   make ui-i18n-extract
   ```
3. Find the new `<trans-unit>` in `src/locale/messages.xlf`
4. Copy it to `src/locale/messages.ja.xlf` and add a `<target>` element:
   ```xml
   <trans-unit id="vulnList.clearFilters" datatype="html">
     <source>Clear filters</source>
     <target>フィルターをクリア</target>
   </trans-unit>
   ```
5. Verify: `make ui-build`

### Updating existing translations

1. Modify the template text (source)
2. Re-extract: `make ui-i18n-extract`
3. Update the `<source>` and `<target>` in `messages.ja.xlf`

## Build Output

Production build generates locale-specific bundles:

```
ui/dist/mayu/
├── en/           # English bundle
│   ├── index.html
│   └── ...
└── ja/           # Japanese bundle
    ├── index.html
    └── ...
```

## Development

During development (`make ui-dev`), the app runs in source locale (English) without localization processing for fast rebuilds.

To test a specific locale during development:
```bash
cd ui && npx ng serve --configuration=production --localize
```

## Adding a New Locale

1. Create `src/locale/messages.{locale}.xlf` (copy from `messages.xlf`)
2. Add translations in `<target>` elements
3. Register in `angular.json` under `i18n.locales`:
   ```json
   "locales": {
     "ja": "src/locale/messages.ja.xlf",
     "fr": "src/locale/messages.fr.xlf"
   }
   ```
4. Build and verify: `make ui-build`
